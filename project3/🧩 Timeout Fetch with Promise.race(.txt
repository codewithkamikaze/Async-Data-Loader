🧩 Timeout Fetch with Promise.race()

🗝️ Introduction
In real-world applications, network requests can sometimes take too long or hang due to slow connections or unresponsive servers. To keep applications responsive, developers often implement timeouts. This lesson explains how to create a manual timeout mechanism using Promise.race() combined with async/await.


🧩 Project Overview
This project demonstrates how to fetch data from an API while enforcing a maximum waiting time. If the request does not finish within the specified timeout, the operation fails immediately and shows an error instead of waiting indefinitely.


🧬 Core Concepts
🔹 Why Fetch Needs Timeouts
The native fetch() API does not provide a built-in timeout option. Without extra logic, a request may stay pending for a long time.

🔹 Timeout Promise
A custom promise is created using setTimeout() that rejects after a given number of milliseconds.

🔹 Promise.race() – Fastest Wins
Promise.race() runs multiple promises at the same time and settles as soon as one promise settles first.
🔹 If fetch() finishes first → data is used
🔹 If timeout() finishes first → request fails

🔹 Reusable fetchWithTimeout() Function
The logic is wrapped inside a helper function so it can be reused for any API call that needs a timeout.

🔹 Error Handling
Both timeout errors and server errors are handled using try / catch, ensuring clean feedback to the user.

🔹 User-Controlled Timeout
The timeout duration is adjustable via an input field, allowing experimentation and better understanding of request behavior.


🔗 Interconnection Between Concepts
🔹 setTimeout() + Promise → Creates a controllable timeout signal.
🔹 Promise.race() + fetch() → Enforces a maximum waiting time.
🔹 async / await → Keeps asynchronous logic readable and sequential.
🔹 try / catch → Unifies timeout and network errors into one flow.
🔹 UI feedback → Shows loading, success, or failure clearly.


🏁 Conclusion
This lesson introduces an essential defensive programming technique: request timeouts. By combining Promise.race() with a custom timeout promise, developers gain full control over how long an application waits for a response. This pattern is widely used in production systems to improve reliability, responsiveness, and user experience.
