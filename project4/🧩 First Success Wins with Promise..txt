🧩 First Success Wins with Promise.any()
🗝️ Introduction
In distributed systems and modern web applications, it is common to have multiple fallback sources for the same data (mirror servers, backup APIs, CDNs). Instead of waiting for all responses or failing on the first error, sometimes the best strategy is to accept the first successful result. This lesson explains how to implement that behavior using Promise.any().


🧩 Project Overview
This project simulates multiple servers responding at different times, where some requests fail and one succeeds. Using Promise.any(), the application automatically selects the first successful response and ignores all failures unless every request fails.


🧬 Core Concepts
🔹 Problem: Redundant Data Sources
When multiple endpoints provide the same data, waiting for all of them is unnecessary and inefficient.

🔹 Promise.any() Behavior
Promise.any() runs multiple promises in parallel and:
🔹 Resolves with the first fulfilled promise
🔹 Ignores rejected promises
🔹 Rejects only if all promises fail

🔹 Simulated Servers
Each “server” is represented by a promise that either:
🔹 Resolves after a delay (success)
🔹 Rejects after a delay (failure)

🔹 AggregateError
If all promises reject, Promise.any() throws an AggregateError, which represents multiple failures grouped into one error.

🔹 UI Feedback Flow
The UI logs each step clearly:
🔹 Starting request
🔹 Winning server
🔹 Total failure scenario


🔗 Interconnection Between Concepts
🔹 Promise.any() → Selects the fastest successful response.
🔹 Parallel execution → All servers are queried at the same time.
🔹 Rejection tolerance → Failures are ignored until no success exists.
🔹 AggregateError handling → Detects complete system failure.
🔹 UI logging → Makes async behavior visible and understandable.


🏁 Conclusion
This lesson demonstrates a powerful resilience pattern: “first success wins.” By using Promise.any(), applications can stay fast and reliable even when some endpoints fail. This approach is widely used in systems that rely on redundancy, mirrors, and fallback services to ensure high availability and better user experience.
