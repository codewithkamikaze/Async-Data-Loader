🧩 Parallel Fetch Dashboard with Promise.all()

🗝️ Introduction
When a page needs multiple pieces of data (for example: users + posts), loading them one after another can make the app slower. A common professional solution is to fetch multiple endpoints in parallel so everything loads faster. This lesson explains how to do that using Promise.all() with async/await, while still handling errors and UI states cleanly.


🧩 Project Overview
This project is a small dashboard that loads users and posts from two API endpoints at the same time. After both responses arrive, it parses the JSON data (also in parallel), then calculates and displays quick statistics in UI cards.


🧬 Core Concepts
🔹 Parallel Requests (Faster Loading)
Instead of waiting for the first fetch to finish before starting the second, both requests start immediately, reducing total loading time.

🔹 Promise.all() for Concurrency
Promise.all([ ... ]) runs multiple promises together and returns results only when all succeed. If one fails, the whole operation fails.

🔹 Two-Phase Parallelism
This project uses parallelism twice:
🔹 Phase 1: Fetch users + posts in parallel
🔹 Phase 2: Parse usersRes.json() + postsRes.json() in parallel

🔹 UI State Management
The UI provides clear feedback:
🔹 Button disabled during loading
🔹 Status pill shows Idle → Loading… → Ready / Error
🔹 Error text displayed only when needed

🔹 Error Handling
The code checks res.ok for both responses. If any request fails, it throws an error and shows a friendly message.

🔹 Simple Local Computation
After data arrives, the dashboard calculates:
🔹 Average posts per user = posts.length / users.length (rounded)


🔗 Interconnection Between Concepts
🔹 Promise.all() + fetch() → Runs multiple network requests in parallel (speed).
🔹 Promise.all() + res.json() → Parses multiple JSON bodies in parallel (efficiency).
🔹 try/catch → Captures any failure from either request or parsing.
🔹 UI state updates → Keeps the user informed and prevents double-click requests.
🔹 Local computations → Turns raw API data into meaningful dashboard insights.


🏁 Conclusion
This project teaches a key real-world frontend pattern: parallel data loading. By combining Promise.all() with async/await, plus clean error handling and UI feedback, you get a faster and more professional dashboard experience—exactly how modern apps load multiple resources efficiently.
